# 首页布局重构设计

## 背景

当前首页默认展示成语选择器，用户需要先选择成语才能看到已有绘本。对于已有绘本的用户，每次打开应用都要先看到选择区而非书架，不够直观。同时已生成的绘本无法重新生成，场景数固定为 6 不够灵活。

## 目标

1. 首页默认展示绘本库，让用户一打开就能看到已有绘本
2. 已生成的绘本支持「重新生成」（删除旧的 + 重新走生成流程）
3. 场景数由 LLM 根据故事复杂度自主决定（5-10 个）

## 设计方案

### 1. 首页布局重构

**当前布局**：左 2/3 成语选择器 + 右 1/3 任务队列

**新布局**：
```
┌──────────────────────────────────────────┐
│  Header: 🎨 成语绘本工坊  | 首页 | 创建   │
├──────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐ │
│  │ 🎭 选择成语 [成语网格] [🔄 换一批]  │ │  ← 紧凑选择区
│  │ ✏️ 输入成语...  [🚀 开始生成绘本]    │ │
│  │ [任务队列: 状态概览]                 │ │
│  └─────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│  📚 我的绘本书架 (N本)                    │
│  ┌──────┐ ┌──────┐ ┌──────┐             │
│  │      │ │      │ │      │             │
│  │ 封面  │ │ 封面  │ │ 封面  │             │
│  │      │ │      │ │      │             │
│  │标题   │ │标题   │ │标题   │             │
│  │阅读🔄删除│ │阅读🔄删除│ │阅读🔄删除│    │
│  └──────┘ └──────┘ └──────┘             │
└──────────────────────────────────────────┘
```

- 上半部分：紧凑的成语选择区（简化版 IdiomSelector）+ 任务队列状态
- 下半部分：绘本库网格（从 `/library` 页面迁移）

### 2. 绘本库卡片（BookCard）

从 `app/library/page.tsx` 抽取为独立组件 `components/BookCard.tsx`：

- 封面图片 + 标题 + 创建日期 + 含义摘要 + 场景数
- 三个操作按钮：📖 阅读 | 🔄 重新生成 | 🗑️ 删除
- 「重新生成」按钮点击后弹出确认框，确认后删除旧绘本并跳转生成流程

### 3. 智能场景数（5-10）

修改 `app/actions/decompose.ts` 中的 prompt 模板：

- 将 `拆分为 6 个关键场景` 改为 `拆分为 5-10 个关键场景`
- 增加指引：简单故事 5-6 个场景，复杂故事 7-10 个场景
- task-executor 动态创建 generate 子任务，自然适配不同场景数

### 4. Header 调整

- 将「📚 绘本库」改为「首页」（链接到 `/`）
- 新增「✏️ 创建」链接（滚动到选择区或链接到 `/#create`）

### 5. /library 路由处理

- `/library` 页面改为重定向到首页（`/`）
- 或者保留 `/library` 但内容与首页绘本库一致

## 文件变更清单

| 文件 | 变更类型 | 说明 |
|------|---------|------|
| `app/page.tsx` | 重构 | 改为绘本库 + 紧凑选择区布局 |
| `components/BookCard.tsx` | 新增 | 抽取 BookCard 为独立组件，增加「重新生成」按钮 |
| `components/IdiomSelector.tsx` | 修改 | 新增 compact 模式 |
| `components/Header.tsx` | 修改 | 调整导航链接 |
| `app/library/page.tsx` | 简化 | 改为重定向到首页 |
| `app/actions/decompose.ts` | 修改 | 场景数从固定 6 改为 5-10 |

## 数据流

1. 用户在首页选择成语 → 跳转 `/generate` → 完成后自动跳转回首页
2. 用户点击「重新生成」→ 确认 → 删除旧绘本 → 跳转 `/generate` → 完成后跳转回首页
3. LLM 根据故事复杂度自主决定场景数（5-10），task-executor 动态创建对应数量的 generate 子任务
